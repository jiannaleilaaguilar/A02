A02 – Git, GitHub, and WebStorm Tutorial

Welcome! These are my instructions for getting started with using Git, GitHub, and WebStorm.
I have made this tutorial easy to follow and hope that it helps you start your journey using Git, GitHub, and WebStorm for all things code related :)
Note: I have included a glossary for key terms used in this document.

Follow these instructions in order:

Creating a Github Account:
Install Github or open Github on your browser of choice using the url: https://github.com
Create an account using your email, username, password, and Country/Providence. A verification link will be sent to your email to confirm your account. 
Installing Webstorm
Download Webstorm for your operating system using the link: https://www.jetbrains.com/webstorm/
Install the program.
Activate with a student license if required.
Installing Git
Download Git for your operating system using the link:  https://git-scm.com/downloads
Ensure download is set to default settings
Verify installation by opening Terminal if on Mac or Command Prompt if on Windows and type: git --version . If installed correctly, the version number will appear.


Creating a Repository        
Once Webstorm is launched, select a New Project. Choose a name associated with your project as this will store your project and any files needed. Ensure the box labeled "Create Git Repository" is checked. 
Right-click the project folder in the left sidebar, select New > HTML File, and name it. Add your code in this file and click the Commit tab in the top-right or left sidebar. Check the box for your html file and type a message like "Initial commit for [Project name], and click Commit.
Go to the top menu and select Git > GitHub > Share Project to connect GitHub.
For future changes you make in this file, it must be "pushed." Click the Push icon to send your local work to the Remote server.


Glossary:
Branch- timeline for your code that allows you to develop features in isolation before adding them to the main project.
Clone- term for downloading an existing Repository from Github to your local computer so you can work on it.
Commit- saves your project at a specific point in time.
Fetch- checks  the remote repository for updates.
GIT- used to track changes in source code during software development.
Github- web-based platform used for version control and hosting GIT repositories.
Merge- process of joining two different branch histories together.
Merge Conflict- where GIT cannot automatically decide which code to keep.
Push- command used to upload your local commit history to a remote repository on Github.
Pull- command that downloads changes from the remote repository and immediately merges them into your local files.
Remote- version of your project hosted on a server 
Repository-directory or storage space where your project, its files, and its entire version history are kept for you to reference.
